Real-time Cryptocurrency Data Processing and Integration

I designed a system that regularly fetches data from the CoinCap API every 5 minutes using a cronjob. The data is then transformed to a more standard time format, and the updated info is stored in a PostgreSQL database.

To make things easier, I set up an automated infrastructure using a Make File. This involves using a Docker Compose file to create images, setting up an AWS EC2 instance with Terraform, and making a Docker image for Metabase. This setup lets us easily visualize the real-time processing of cryptocurrency data in Metabase dashboards.

GitHub: https://github.com/Yogendra-sodha/Real-time-Cryptocurrency-Data-Processing-and-Integration

First Click Attribution E-commerce Streaming Pipeline

Developed a Python script to create synthetic E-commerce data, encompassing user interactions, clicks, and checkout details. This data was then directed into Kafka topics, with consumers subscribing to ingest the information into a Flink cluster for computation.

Employed the Kafka JDBC connector to consume data from the Kafka topic, executing real-time processing within the Flink cluster based on a "first click" condition. The processed data was subsequently loaded into a PostgreSQL sink table, ensuring seamless storage of the computed results.

Implemented Apache Flink's interval and time-based processing join to perform a left join between checkouts and clicks tables within the past hour. This approach ensures accurate attribution with minimal state retention. Utilized HashMap as the state backend to efficiently store larger datasets while maintaining low-latency read times.

GitHub: https://github.com/Yogendra-sodha/First-Click-Attribution-E-commerce-Streaming-Pipeline
